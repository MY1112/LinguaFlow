# AGENTS.md

## 项目说明

LinguaFlow 是一个基于 Python + PySide6 + llama.cpp 的本地 AI 桌面工具。

本项目遵循 Local First 与 Feature First 的设计理念。

所有核心能力默认本地运行，不依赖云服务。

本项目当前目标仅实现 V1 功能，不提前开发未来规划功能。

---

# 必须阅读的文档

开始任何开发之前，必须阅读：

docs/architecture/00-开发原则.md

docs/architecture/01-产品需求(PRD).md

docs/architecture/02-工程架构.md

docs/architecture/03-项目结构.md

docs/architecture/04-开发规范.md

如果文档之间存在冲突，不允许自行决定。

停止开发，并向用户报告。

---

# 开发原则

严格遵守 architecture 文档。

禁止：

修改项目架构。

禁止：

新增顶层目录。

禁止：

跨 Feature 调用。

禁止：

提前开发未安排的功能。

禁止：

过度设计。

优先保持代码简单、清晰、可维护。

---

# 工作方式

开发采用 Task 驱动。

一次只允许开发一个 Task。

不要开发当前 Task 之外的内容。

不要提前实现未来功能。

---

# 开发流程

收到 Task 后：

1. 阅读相关文档。

2. 理解 Task。

3. 修改代码。

4. 自检。

5. 汇报修改内容。

不要跳过任何步骤。

---

# 修改范围

只允许修改当前 Task 涉及文件。

不要顺便重构其它模块。

不要因为觉得更合理而修改架构。

---

# 代码要求

遵循 docs/architecture/04-开发规范.md。

所有代码：

- 可读
- 易维护
- 单一职责
- 不过度封装

---

# 自检

完成开发后必须检查：

项目是否能够运行。

是否违反开发规范。

是否新增循环依赖。

是否新增无用代码。

是否存在明显 Bug。

# 遇到以下情况必须停止

文档冲突。

需求不明确。

需要修改架构。

需要新增顶层目录。

需要删除已有模块。

需要修改开发原则。

停止并等待用户确认。

不要猜测实现。
