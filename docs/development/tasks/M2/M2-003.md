# M2-003 ModelService

## 目标

建立整个项目唯一的模型业务层（ModelService）。

ModelService 负责管理模型生命周期，并向上层提供统一的推理接口。

它屏蔽底层 llama.cpp Adapter，实现业务层与推理引擎解耦。

---

## 职责

负责：

- 加载模型
- 卸载模型
- 判断模型是否已加载
- 文本推理
- 管理默认推理参数

不负责：

- Prompt 管理
- 翻译业务
- OCR
- UI
- 全局快捷键

---

## 目录位置

```
src/services/model_service.py
```

---

## 对外接口

至少提供：

```python
load_model(model_path: str)

unload_model()

is_loaded() -> bool

generate(
    prompt: str,
    *,
    temperature: float = 0.2,
    top_p: float = 0.9,
    max_tokens: int = 512,
) -> str
```

---

## 设计原则

ModelService 是唯一允许调用 LlamaCppAdapter 的业务层。

禁止：

```
Feature
    ↓
Adapter
```

必须：

```
Feature
    ↓
ModelService
    ↓
Adapter
```

---

## 返回值要求

generate() 必须返回：

```python
str
```

禁止返回：

- Completion
- Llama
- 第三方对象

所有第三方对象必须在 Adapter 内完成转换。

---

## Runtime

Application 启动时创建 ModelService。

退出程序时：

```
RuntimeManager
    ↓
ModelService
    ↓
Adapter
```

释放模型资源。

---

## 日志

统一使用 Logger。

记录：

- 模型加载
- 模型卸载
- 推理开始
- 推理结束
- 异常

禁止使用 print()。

---

## 异常

新增：

```
ModelServiceError
```

禁止直接向上抛出：

```
LlamaCppAdapterError
```

---

## Out Of Scope

本任务禁止实现：

- Prompt
- 翻译
- OCR
- UI
- Tray
- Hotkey
- Clipboard
- Screenshot

---

## Definition of Done

完成后必须满足：

- ModelService 能加载真实 GGUF 模型
- generate() 完成一次真实推理
- generate() 返回 string
- RuntimeManager 退出时正确释放模型
- Ruff 通过
- Black 通过
- Python 语法检查通过
- python -m src.main 启动通过
