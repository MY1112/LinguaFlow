# LinguaFlow 项目结构

> 版本：v1.0

本文档说明整个项目的目录结构及各目录职责。

---

# 一、项目目录

```
LinguaFlow/

├── docs/
├── models/
├── config/
├── logs/
├── resources/
├── src/
├── tests/
├── pyproject.toml
└── README.md
```

---

# 二、docs/

项目文档。

包括：

```
00-开发原则.md

01-产品需求(PRD).md

02-工程架构.md

03-项目结构.md

04-开发规范.md

05-Prompt规范.md

06-Codex开发指南.md

07-发布计划.md

08-技术选型.md
```

---

# 三、models/

本地模型目录。

例如：

```
Qwen2.5-3B-Instruct.gguf

Qwen2.5-7B-Instruct.gguf

DeepSeek-R1.gguf
```

程序默认从这里加载模型。

---

# 四、config/

用户配置。

例如：

```
settings.json

hotkey.json
```

程序运行时自动读取。

---

# 五、logs/

日志输出目录。

例如：

```
2026-07-22.log
```

统一由 Logger 管理。

禁止手动写日志文件。

---

# 六、resources/

公共资源。

例如：

```
icons/

images/

fonts/

themes/
```

Feature 专属资源不要放这里。

---

# 七、src/

项目源码。

```
src/

app/

ui/

features/

services/

integrations/

runtime/

core/

main.py
```

---

# 八、app/

负责程序生命周期。

例如：

```
application.py
```

这里只负责：

程序初始化。

---

# 九、ui/

公共界面。

例如：

```
main_window.py

tray.py

dialogs/

components/
```

这里只放：

多个 Feature 共用的界面。

---

# 十、features/

项目核心。

所有业务都放这里。

例如：

```
translate/

ocr/

writing/

summary/

history/

settings/
```

以后：

新增功能。

直接新增 Feature。

---

# 十一、Feature 模板

建议保持统一。

```
feature_name/

api.py

service.py

prompt.py

models.py

manifest.py

widgets/

resources/
```

其中：

manifest.py

描述：

- Feature 名称
- Feature 版本
- Feature 描述

用于后续扩展。

---

# 十二、services/

共享服务。

例如：

```
model_service.py

config_service.py

prompt_service.py

history_service.py
```

不要写：

```
translate_service.py
```

因为：

Translate 属于 Feature。

---

# 十三、integrations/

第三方封装。

例如：

```
clipboard.py

hotkey.py

ocr.py

llama_cpp.py
```

禁止：

UI 直接调用第三方。

---

# 十四、runtime/

程序运行状态。

例如：

```
app_context.py

model_manager.py

thread_pool.py

task_manager.py
```

这里管理：

程序生命周期中的状态。

---

# 十五、core/

公共能力。

例如：

```
logger.py

event_bus.py

constants.py

exceptions.py

utils.py
```

所有模块都可以使用。

---

# 十六、tests/

测试代码。

目录建议与 src 保持一致。

例如：

```
tests/

translate/

services/

integrations/
```

---

# 十七、README.md

项目说明。

包含：

- 项目介绍
- 安装方法
- 快速开始
- 常见问题

---

# 十八、目录规范

新增目录前，应优先确认：

是否可以放入：

- Feature
- Service
- Integration

避免新增新的顶级目录。

保持工程结构稳定。
