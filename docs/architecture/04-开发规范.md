# LinguaFlow 开发规范

> 版本：v1.0

本文档规定 LinguaFlow 项目的编码规范、模块职责和开发流程。

所有开发必须遵循本规范。

---

# 一、总体原则

代码应该：

- 简单
- 可读
- 易维护
- 易测试

不要为了"高级设计"增加复杂度。

---

# 二、Feature 规范

每一个功能都是一个 Feature。

建议结构：

```
translate/

api.py

service.py

prompt.py

models.py

manifest.py

widgets/

resources/
```

新增功能：

新增 Feature。

不要修改已有 Feature。

---

# 三、Feature 的职责

api.py

负责：

- 接收 UI 请求
- 调用 Service
- 返回结果

禁止：

- 编写业务逻辑
- 操作数据库
- 调用第三方 SDK

---

service.py

负责：

真正业务逻辑。

例如：

翻译。

Prompt 拼接。

结果处理。

允许：

调用：

Services

Integrations

Core

禁止：

更新 UI。

---

prompt.py

负责：

Prompt 模板。

禁止：

出现 Python 业务逻辑。

---

models.py

负责：

数据对象。

例如：

Request

Response

Settings

禁止：

业务逻辑。

---

widgets/

负责：

Feature 自己的 UI。

不要把 Feature UI 放到：

ui/

---

manifest.py

描述：

Feature 信息。

例如：

名称。

版本。

描述。

---

# 四、Services

这里只放：

共享能力。

例如：

```
ModelService

ConfigService

PromptService

HistoryService
```

禁止：

出现：

TranslateService。

---

# 五、Integrations

这里只封装第三方。

例如：

llama.cpp

PaddleOCR

Clipboard

Hotkey

Screenshot

禁止：

业务判断。

例如：

OCR：

不能：

自动翻译。

---

# 六、UI

UI：

只负责：

展示。

所有业务：

交给 Feature。

禁止：

直接调用：

llama.cpp。

---

# 七、Runtime

负责：

程序运行状态。

例如：

模型。

线程。

任务。

不要写业务。

---

# 八、Core

这里只放：

公共能力。

例如：

Logger

Utils

Constants

EventBus

---

# 九、异常处理

禁止：

忽略异常。

必须：

统一记录日志。

必要时：

向 UI 返回友好错误。

---

# 十、日志规范

禁止：

print()

统一：

Logger。

日志等级：

DEBUG

INFO

WARNING

ERROR

CRITICAL

---

# 十一、异步任务

耗时任务：

必须后台执行。

例如：

模型加载。

OCR。

推理。

禁止阻塞 UI。

---

# 十二、配置

禁止：

直接读取：

settings.json

统一：

ConfigService。

---

# 十三、Prompt

Prompt：

属于 Feature。

例如：

translate/

prompt.py

不要建立：

prompt_center.py

---

# 十四、代码规模建议

函数：

≤50 行

类：

≤300 行

文件：

≤300 行

超过建议拆分。

---

# 十五、命名规范

文件：

snake_case

类：

PascalCase

变量：

snake_case

常量：

UPPER_CASE

---

# 十六、新增 Feature 流程

1.

创建目录。

2.

创建：

api.py

service.py

prompt.py

models.py

manifest.py

3.

实现业务。

4.

实现 UI。

5.

编写测试。

6.

更新文档。

---

# 十七、禁止事项

禁止：

God Object

God Service

循环依赖

复制 Prompt

全局变量

跨 Feature 调用

UI 写业务

Integration 写业务

---

# 十八、最终目标

任何开发者看到一个 Feature。

都应该：

一分钟内理解。

十分钟内修改。

不需要阅读整个项目。
